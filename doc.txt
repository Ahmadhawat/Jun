No, you do not need to edit compose.yml every time.

Make one compose file with 4 model profiles. Then you start only the model you want.

1. Create folders

sudo mkdir -p /home/vllm/models
sudo chown -R $USER:$USER /home/vllm
cd /home/vllm

2. Create /home/vllm/compose.yml

services:
  gpt-oss:
    image: vllm/vllm-openai:latest
    container_name: vllm-gpt-oss
    profiles: ["gpt-oss"]
    runtime: nvidia
    ipc: host
    ports:
      - "8000:8000"
    volumes:
      - /home/vllm/models:/root/.cache/huggingface
    command: >
      --model openai/gpt-oss-20b
      --served-model-name gpt-oss
      --host 0.0.0.0
      --port 8000
      --gpu-memory-utilization 0.90
      --max-model-len 8192

  granite:
    image: vllm/vllm-openai:latest
    container_name: vllm-granite
    profiles: ["granite"]
    runtime: nvidia
    ipc: host
    ports:
      - "8000:8000"
    volumes:
      - /home/vllm/models:/root/.cache/huggingface
    command: >
      --model PUT-GRANITE-AWQ-OR-GPTQ-REPO-HERE
      --served-model-name granite
      --host 0.0.0.0
      --port 8000
      --quantization awq
      --gpu-memory-utilization 0.90
      --max-model-len 8192

  gemma:
    image: vllm/vllm-openai:latest
    container_name: vllm-gemma
    profiles: ["gemma"]
    runtime: nvidia
    ipc: host
    ports:
      - "8000:8000"
    volumes:
      - /home/vllm/models:/root/.cache/huggingface
    command: >
      --model pytorch/gemma-3-27b-it-AWQ-INT4
      --served-model-name gemma
      --host 0.0.0.0
      --port 8000
      --quantization awq
      --gpu-memory-utilization 0.90
      --max-model-len 8192

  qwen:
    image: vllm/vllm-openai:latest
    container_name: vllm-qwen
    profiles: ["qwen"]
    runtime: nvidia
    ipc: host
    ports:
      - "8000:8000"
    volumes:
      - /home/vllm/models:/root/.cache/huggingface
    command: >
      --model Qwen/Qwen3-32B-AWQ
      --served-model-name qwen
      --host 0.0.0.0
      --port 8000
      --quantization awq
      --gpu-memory-utilization 0.90
      --max-model-len 8192

openai/gpt-oss-20b, pytorch/gemma-3-27b-it-AWQ-INT4, and Qwen/Qwen3-32B-AWQ are Hugging Face model repos that mention vLLM or AWQ usage. Granite 4.1 30B exists as a full IBM repo, but you should replace the placeholder with a real AWQ/GPTQ Granite repo if you find one. 

3. Start one model

GPT-OSS:

docker compose --profile gpt-oss up -d

Gemma:

docker compose --profile gemma up -d

Qwen:

docker compose --profile qwen up -d

Granite:

docker compose --profile granite up -d

4. Switch model

Stop current one:

docker compose down

Start another:

docker compose --profile qwen up -d

5. RAG API

Your RAG always calls the same endpoint:

http://SERVER_IP:8000/v1/chat/completions

Only the model name changes:

gpt-oss
granite
gemma
qwen

So the workflow is:

one compose.yml
four model profiles
one active model at a time
same /home/vllm/models cache
no redownload after first download
